Did the project meet the brief?

My final system met the objectives of the project brief. My embedded system collected environmental data including temperature and soil moisture and simulated logging events using a digital input. The collected data was transmitted to a python program where it was stored in a CSV file and the data was analysed to calculate the risk of a wildfire starting. My system allows for a range of what-if scenarios to be tested and shows how the system will adapt to these changes in conditions.
I think my system could definitely be accurate enough for a real forest, however my system is on a simplified level. I think that the factors being recorded are exactly what needs to be recorded in the event of predicting a wildfire. There is a lack of research on the effect of logging and I think this is the most important factor to research hence why in my project I focused on the factors that were linked to logging directly.

Strentghs:

  • Collected data in real time
  • Predictive modelling
  • Adaptive feedback mechanism (alarms and LEDs)
  • Clear demonstration of wildfire risk
  • Each condition links with each other

Limitations:

  • Limited dataset (as there is only three factors being demonstrated)
  • Inaccurate data being transferred from microbit to python
  • Simple logging simulation

Future Iteration:

  • If I had more time I would try to improve the accuracy of the system by potentially adding a humidity sensor or a wind sensor as I feel this factor would link very well with my other factors. Adding another condition would make the project even more realistic and is definitely something I wish I could've added.
  • I would've liked to try and compare the data that I had collected with real life sources to improve the accuracy of my system however due to the limited time I felt it was more important for me to focus on my own collected data.
